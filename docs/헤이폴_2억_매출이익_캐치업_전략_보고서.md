## 1. Executive Summary

### 핵심 메시지

> **틸리언프로의 현실을 인정하고, 헤이폴을 B2C 리워드앱에서 "데이터 수집 플랫폼"으로 격상시키는 구조적 전환을 추진하되, 전환 과정에서 기존 자산을 활용한 즉시 매출을 병행하여 연내 2억원 매출이익을 달성합니다.**

### 요약 수치

| 구분 | 금액 | 비중 |
|------|------|------|
| **Track A (즉시 매출)** — 기존 채널 극대화 | **1.18억원** | 59% |
| **Track B (구조적 매출)** — 신규 B2B 기능 | **0.92억원** | 41% (하반기 가속) |
| **합계** | **2.10억원** | 105% 달성 |

### 왜 이중 트랙인가

- **Track A만 진행 시**: 단기 매출은 확보하나, 내년에도 같은 고민 반복 → 지속 불가능
- **Track B만 진행 시**: 올해 매출 공백 발생, 그룹 목표 미달 → 실행력 의심
- **이중 트랙**: 올해 목표 달성 + 내년부터 가속 성장 기반 확보 → 최적 균형

| 항목 | 현황 |
|------|------|
| 플랫폼 연령 | 10년+ 미개편 |
| 3년 사외매출 | 약 2.5억원 (연평균 8,300만원) |
| 3년 건수 | 1,029건 (건당 평균 24만원) |
| 경쟁 환경 | 오픈서베이(패널 700만+), 엠브레인(150만 패널), 서베이몽키 등 상향 평준화 |
| UI/UX | 모바일 미최적화, 현대 셀프서베이 플랫폼 대비 기능·디자인 열위 |
| 영업 현실 | 기존 고객 유지 중심, 신규 영업 시 경쟁 열위로 수주 어려움 |

**판단: 틸리언프로 개편 없이 매출 증대는 비현실적. 기존 고객 유지 + 인바운드 대응 수준으로 운영.**

### 2-2. 헤이폴: 왜 여기에 집중하는가

| 항목 | 현황 |
|------|------|
| 리팩토링 | 2025.12.24 신버전 런칭 완료 |
| 인프라 | Kubernetes, ElasticSearch, Redis, AWS, CI/CD 파이프라인 |
| AI 역량 | Naver HyperCLOVA X 연동 (AI 설문 설계·보고서 자동화) |
| 패널 규모 | 38,000명 활성 패널, DAU 28,000~29,000명 |
| 프로필 데이터 | 11개 카테고리, 100+ 설문 템플릿 |
| 현재 한계 | B2C 전용 (패널 참여·리워드만 존재), B2B 기능 부재 |

**판단: 최신 인프라 + AI 역량 + 활성 패널을 보유. B2B 기능만 추가하면 즉시 시장 진입 가능.**

### B1. 외부 폼 연동 서비스 (5월 런칭)

**개념**: 구글폼, 네이버폼, 타입폼 등 외부 설문 URL을 헤이폴에 등록하면, 헤이폴 패널이 참여하고 완료 인증 → 과금

| 항목 | 수치 |
|------|------|
| 타겟 고객 | 대학원생, 스타트업, 마케터 |
| 과금 모델 | 응답 건당 1,000~2,500원 (타겟팅 정도에 따라) |
| 패널 보상 | 건당 300~800원 (CO 기준) |
| 플랫폼 마진 | 건당 700~1,700원 (약 60~70%) |
| 5~12월 목표 | 월 평균 80건 등록 × 평균 50응답 = 4,000응답/월 |
| **8개월 매출이익** | **2,240만원** |

> 핵심: 학술 연구자 세그먼트는 "패널 모집"이 최대 고통점. 구글폼+패널 연동만으로 즉시 수요 흡수 가능.

### B2. 셀프 프로젝트 등록 (6월 런칭)

**개념**: 기업/개인이 헤이폴 자체 설문 빌더로 프로젝트를 직접 등록·배포·결과 확인

| 항목 | 수치 |
|------|------|
| 타겟 고객 | SMB, 스타트업, 마케팅팀 |
| 과금 모델 | 응답 건당 1,500~3,000원 |
| 패널 보상 | 건당 500~1,000원 |
| 플랫폼 마진 | 건당 1,000~2,000원 |
| 6~12월 목표 | 월 평균 30건 × 평균 100응답 |
| **7개월 매출이익** | **2,100만원** |

### B3. 참여자 모집 서비스 — IDI/FGI (7월 런칭)

**개념**: 심층 인터뷰·포커스 그룹 참여자를 프로필 기반으로 모집·매칭

| 항목 | 수치 |
|------|------|
| 타겟 고객 | UX 리서치팀, 컨설팅펌, 대기업 상품기획 |
| 과금 모델 | 참여자 1인당 30,000~80,000원 (모집비) |

## 10. 선순환 구조: 왜 이 전략이 지속 가능한가

### 10-1. 유저 유입 선순환

```
                    ┌──────────────┐
                    │ 학술 연구자    │
                    │ (외부 폼 연동) │
                    └──────┬───────┘
                           │ 헤이폴에 패널 모집 의뢰
                           ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ UX 리서처     │───▶│   헤이폴      │◀───│ 마케터        │
│ (IDI/FGI 모집)│    │  B2B 플랫폼   │    │ (셀프 서베이)  │
└──────────────┘    └──────┬───────┘    └──────────────┘
                           │
                    패널에게 다양한 설문 제공
                           │
                    ┌──────▼───────┐
                    │ 패널 만족도 ↑  │
                    │ CO율 ↑        │
                    │ 리텐션 ↑      │
                    │ MAU ↑         │
                    └──────┬───────┘
                           │
                    더 많은 패널 = 더 빠른 데이터 수집
                           │
                    ┌──────▼───────┐
                    │ B2B 고객 만족 ↑│
                    │ 재구매 + 추천  │
                    │ 매출 성장      │
                    └──────────────┘
```

### 10-2. 수요-공급 선순환

**지금의 문제**: 패널 38,000명이 있는데 설문 공급이 부족 → 패널이 돈을 못 벌어서 이탈

**전략 적용 후**:
1. 외부 폼 연동 → 설문 공급 즉시 증가 (구글폼만 등록하면 됨)
2. CPA 광고 → 설문 없을 때도 패널이 돈 버는 채널 확보
3. 설문 공급 증가 → CO율 상승 (참여할 설문이 많으니까)
4. CO율 상승 → 패널 만족도 상승 → 리텐션 상승
5. 패널 증가 → B2B 고객에게 "패널 규모" 영업 가능
6. B2B 매출 증가 → 설문 공급 더 증가 → **선순환**

### 10-3. 직군 확장 선순환

```
Year 1 (2026):
  학생/연구자 → "패널 모집이 싸고 빠르네" → 헤이폴 이용

Year 2 (2027):
  그 학생이 취업 → UX 리서처, 마케터, 데이터 분석가가 됨
  → 회사에서도 헤이폴 이용 → B2B 기업 고객 전환

Year 3 (2028):
  기업 고객 증가 → 프리미엄 서비스 (구독형, AI 보고서)
  → ARPU 상승 → 매출 가속 성장
```

> **핵심**: 오늘의 학생 유저가 내일의 기업 고객이 됩니다. 학술 연구자 세그먼트는 가장 저렴하게 확보할 수 있으면서, 가장 높은 장기 LTV를 가진 고객군입니다.
